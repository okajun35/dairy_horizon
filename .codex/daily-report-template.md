# Dairy Horizon 日報テンプレート

作業日ごとに `docs/daily-reports/YYYY-MM-DD.md` を作成または更新する。
実装、調査、設計変更のいずれも、作業終了時に短く記録する。事実と判断を分け、未確認事項を成果として扱わない。

```md
# 日報 — YYYY-MM-DD

## 今日の目的

- 今日、何を前に進める日だったか。

## やったこと

- 実施した変更・確認
- 対象ファイル、画面、計算、データ、コミットなど

## 結果・確認できたこと

- 実際にどう動くようになったか
- 実行したテスト・画面確認
- まだ確認できていないこと

## 良かったこと

- 判断・実装・進め方のうち、続けるべきこと

## 良くなかったこと・リスク

- 不足、手戻りの芽、曖昧な前提、品質上の懸念

## 改善すること

- 上の問題に対して次に取る、具体的な行動

## 仮定・出典の変化

- `user_input`、`industry_guidance`、`official_observation`、`derived`、`demo_assumption` の追加・変更
- 変更がなければ「なし」

## 判断待ち・確認したいこと

- ユーザーまたは専門家の判断が必要な事項
- なければ「なし」

## 明日やること

最優先:
- [ ] 一番重要な作業

余力があれば:
- [ ] 二番目の作業

完了条件:
- 明日何ができていれば完了か。
```

毎回、次も一言で確認する。

- この変更で、農家が何を一つ理解・確認できるようになったか。
- GPTと決定論的計算の境界は守られているか。
- 今回、意図して実装しなかったものは何か。
