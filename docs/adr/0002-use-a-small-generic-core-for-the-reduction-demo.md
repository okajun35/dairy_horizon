# ADR-0002: 小さな汎用コアで減頭デモ一本を完成させる

- Status: Accepted
- Date: 2026-07-17

## Context

Dairy Horizonには、現在の不足、3つの設備経路、標準財務、THI背景、AI説明がすでにある。ここへ将来の牛群規模、複数機器、実測補正を同じ深さで追加すると、組み合わせが増え、根拠の薄い表示と未完成の画面が増える。一方、60頭から5年後45頭へ減頭するデモだけを数値でハードコードすると、入力を変えられず、決定論的計算を重視する現在の設計とコード審査の双方に耐えない。

## Decision

### 1. 実装の深さ

実装は次の三層に限定する。

1. 現在と将来の対策対象頭数を受け取る小さな純粋計算
2. 現状維持、第1期、現在頭数の目安まで追加を比較する一つのユースケース
3. 千葉市・現在60頭・既存10台・5年後45頭・第1期5台を主経路とするUI

60頭、45頭、10台、5台はデモ入力であり、計算関数内の条件分岐にはしない。維持と増頭は同じ計算を通し、自動テストで成立を確認する。

### 2. 二時点の境界

現在と5年後を一つの頭数へ混ぜない。最低限、次を別々に返す。

- 現在・追加前の過不足
- 現在・第1期追加後の過不足
- 5年後・第1期追加後の過不足
- 移行期間中に頭数基準上の不足が残るか

年ごとの減頭数と5年間累積ROIは計算しない。「移行期間中の不足」は、精密な年次値ではなく、現在追加後に不足があり将来状態へ移ることを示す説明である。

### 3. 3つの設備経路

減頭デモでは、第1期5台と5年後45頭の頭数目安がともに合計15台になる。そのため比較を重複させず、次を表示する。

- 現状維持: 既存10台
- 第1期: 5台追加して合計15台
- 現在60頭の頭数目安まで追加: 10台追加して合計20台

3案目は、現在の不足を頭数基準上解消する上限比較であり、推薦ではない。5年後45頭なら頭数基準より5台多くなることを示す。

### 4. カバーの状態

3頭／台は `industry_guidance` であり、実際の送風実績ではない。標準型でも次を分ける。

- 頭数基準による新規カバー想定頭数
- 実測で確認できたカバー頭数

実測前の回収条件は `guidance_estimate`、実測頭数を反映した回収条件は `confirmed_measurement` とする。風速条件は送風範囲の確認であり、乳量効果や投資成立の証明とは扱わない。

### 5. 機器種類

標準100cm級だけを頭数基準と収支分岐点まで接続する。省電力100cm級と大型高風量型は、確認できたメーカー仕様から電力費を計算できる場合だけ表示し、必要台数、カバー頭数、1頭当たり回収条件は `needs_measurement` とする。大型ファンの台数を標準型へ自動換算しない。

初回の比較仕様は、利用者提供の「乳牛の暑熱対策チャレンジ ガイドブックin十勝」資材一覧に記載された値から、省電力100cm級を0.25kW／台、大型高風量型を1.055kW／台とする。省電力型5台と大型型2台は `demo_assumption` であり、資料が推奨する必要台数とは扱わない。資料の発行主体と各数値の一次メーカー資料は未確認のため、画面では `manufacturer_spec` と利用者提供資料を分けて表示し、価格、施工費、送風範囲を補完しない。

### 6. 次の一問

決定論的コードが未確認条件を次の順で返す。

1. 将来頭数が未入力なら将来の対策対象頭数
2. 送風範囲が未確認なら牛体付近風速
3. 送風範囲確認後は夏季乳量差

AIは計算済み結果と候補を受け取り、利用者向けの一問として説明する。AI自身が台数、費用、感度を計算しない。API失敗時は同じ決定論的結果をテンプレートで表示する。

## Consequences

### Positive

- 一本のデモを完成させながら、維持・増頭にも同じ計算を使える
- 現在の不足と将来の不足を混同しない
- 「未評価」をTODOではなく、根拠不足を表す正式な結果にできる
- AIが必要な理由を、次の一問の説明として示せる

### Trade-offs

- 移行期間の年次頭数と累積ROIは分からない
- 標準型以外は同じ深さの回収比較ができない
- 風速を確認しても、夏季乳量差がなければ経済効果は確定しない

## Rejected alternatives

- 60頭から45頭の数値を画面と計算へ直接ハードコードする
- 3牛群方針×3設備経路×3機器を27枚のカードで表示する
- 全機器を根拠のない換算で同じ回収計算へ接続する
- 年次最適化、精密キャッシュフロー、AIによる設備自動推薦まで広げる
