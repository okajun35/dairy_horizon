# ADR-0006: THIの暑熱負荷と送風能力を分け、段階的な適応経路モデルへ進む条件を定める

- Status: Accepted
- Date: 2026-07-18

## Context

将来のTHI 72以上の日数が増えることは、現在の暑熱対策が必要になる日数、
運転時間、電気代、保守負担が増える可能性を示す。しかし、日数だけから
「ファンを何台追加するか」や「何年に導入するか」は決まらない。

同じ日数でも、ピークTHI、連続時間、夜間に回復できるか、牛群の泌乳段階、
牛体付近の風速、既存ファンの配置・角度・清掃状態によって牛の熱負荷は異なる。
THIは気温と湿度から決まる環境指標であり、ファンがTHI自体を書き換えるものではない。

現在のPhase 1には、期間単位のTHI 72以上日数と、頭数基準のファン台数、
牛床への均等配置による表示用推計がある。時間別の暑熱負荷、牛床ごとの実測風速、
牛群リスク区分、国内農場で検証された台数決定式はない。

## Decision

### 1. Phase 1では、THIを送風能力へ直接変換しない

Phase 1は次を表示する。

- 地域の現在相当と将来期間のTHI 72以上日数、および現在との差
- 2035年以降など取得していない期間は「未取得」
- 頭数基準による現在の不足と、現在・第1期・全数追加の比較
- 牛体付近風速、夏季乳量差など、次に現場で確認する一件

画面では、頭数と設備条件が同じなら現在の不足が自然には解消せず、
THI対象日数が増えると、その不足が問題になる日数と運転負担は増える可能性を明示する。

ただし、次をPhase 1で出力しない。

- THI対象日数だけからの追加ファン台数
- THI対象日数だけからの投資実施年
- 将来気候だけからの「カバー不能」判定
- 取得していない年のTHI値

### 2. 「配置上のカバー」と「暑熱対策としての十分性」を分ける

配置上のカバーは、既存の頭数基準と表示用の牛床配置から得る推計である。
同じ台数・同じ配置なら、この推計は時間経過だけでは変わらない。

暑熱対策としての十分性は、将来の時間別暑熱負荷、牛体位置の風速、牛群の状態を
合わせて初めて評価できる。Phase 1ではこれを未評価として扱い、現場確認へつなぐ。

### 3. Phase 2の最小モデルは「時間別暑熱負荷＋風速カバー」とする

Phase 2へ進むときの最初の実装対象は、熱収支モデルや自動最適化ではなく、次に限定する。

```text
時間別THI
＋ 閾値を超えた強さ・継続時間
＋ 牛床ごとの実測または検証済み推定風速
＋ 明示した牛群リスク区分
↓
現在設備で風速・暑熱負荷の確認が必要な牛床と時間帯
↓
配置改善、追加候補、観測トリガーを参考シナリオとして表示
```

出力は「追加2台を確定」ではなく、次のような条件付きのスクリーニングにする。

- 不足する可能性が高い牛床・時間帯
- 配置、角度、清掃を確認する優先順位
- 実測後も不足する場合の追加候補
- 検討開始時期の範囲と、前倒し・延期の観測トリガー

牛体風速の閾値は、現在のPhase 1で使用する2m/s以上の現地確認と混在させない。
別の閾値を導入する場合は、対象牛群、測定方法、出典、国内での扱いを明示してから採用する。

### 4. Phase 2へ進むための入口条件

次のすべてが揃うまで、Phase 2の数値結果を主画面へ出さない。

1. 時間別の気温・湿度を持つ観測・将来気候データと、欠測の表示方針
2. 牛床単位の風速を入力・保存する方法、または検証済みの風速推定方法
3. 追加候補の風速効果を重複なしに扱う根拠
4. 牛群リスク区分と閾値の出典、利用者が確認・変更する方法
5. 少なくとも一農場で、モデル出力と現場の風速・牛の状態を照合する検証計画
6. 「参考シナリオ」「未評価」「実測で確定」の表示規約とテスト

### 5. 将来の適応経路はトリガーを示し、確定年を推薦しない

将来気候は単一予報ではなく複数シナリオの範囲を使う。
出力は「2031年に必ず追加」ではなく、検討開始時期の範囲と次の観測トリガーとする。

- 牛体付近風速の不足
- 呼吸数、体温、反芻、乳量、採食量など、利用可能で根拠を示せる確認値
- 夜間回復や連続暑熱の指標
- 清掃・角度調整後も残る風速不足

## Consequences

### Positive

- 気候の変化を無視せず、根拠のない台数・導入年の自動推薦も避けられる。
- Phase 1の少ない入力と説明可能性を維持できる。
- Phase 2で必要なデータ、検証、出力の境界が明確になる。
- AIは複数の確認値を農家向けに整理できるが、数値モデルの代替にはならない。

### Trade-offs

- Phase 1だけでは、将来の時間別暑熱負荷から追加台数を出せない。
- Phase 2には、データ整備、現場測定、出典確認、検証のコストが必要になる。
- 初期の地域拡張や熱収支モデルは、Phase 2の最小モデルが検証されるまで後回しになる。

## Rejected alternatives

- `THI 72以上の日数が○日増えたらファンを○台追加` という固定ルール
- 取得していない2035年以降のTHIを外挿して投資年を出す方法
- 実測風速なしに、台数だけで牛床ごとの冷却能力を確定する方法
- 高度な熱収支モデル、CFD、自動最適化をPhase 2の最初から同時に導入する方法
