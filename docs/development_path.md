# Dairy Horizon 開発方針と機能追加の道筋

## 目的

Dairy Horizonは、投資判断や見積依頼を自動化する画面ではない。

> 農家が細かい数字を知らなくても、今の牛舎で何が不足しているか、小さく始めるならどの状態を比較するか、次に何を確認すべきかを理解できる適応投資経路ナビゲーター

を目指す。

## 現在の導線

1. 地域、搾乳牛頭数、牛床列数、既存ファン数を入力する。
2. 牛舎で、カバー済み牛と未カバー推計牛を確認する。
3. 現在、第1期、全数整備を切り替えて比較する。
4. 第1期と頭数目安案の導入費、年間電気代、回収に必要な防止乳量を比較する。
5. 次に現場で確認すべきことを一つ表示する。
6. 画面末尾で、標準仮定と出典区分を確認する。

主画面では投資判断、見積依頼、投資年の自動推薦を行わない。

## 第1期の扱い

第1期の5台は、技術基準や最適解ではない。利用者が一度に全数整備を考えず、比較を始めるための初期モデルケースである。

- 初期値は `min(5, 不足台数)` とする。
- 結果画面で、0台から不足台数まで変更可能にする。
- 5台は `demo_assumption` と表示する。
- 変更時には、追加ファンと新規カバー推計牛を牛舎上で更新する。

一方、全酪連資料の「牛体付近で風速2m/s以上」「3頭につき1台以上」は `industry_guidance` として扱う。

## データの利用方針

Webリクエスト中に外部APIを呼ばない。事前に保存・検証したデータだけを読む。

| データ | 用途 | 用途外 |
|---|---|---|
| 千葉の2020〜2025年実測THI | 直近観測基準と現在との差の説明 | 年間台数や投資年の決定 |
| 同一CMIP6モデルの2020〜2025年 | 将来期間とのモデル別変化量 | 過去の実測値としての表示 |
| +1℃／+2℃のデモ暑熱ストレス | 暑熱負荷の説明 | 将来の確定予報 |
| 2025〜2034年CMIP6複数モデル | 5年単位の背景と不確実性の説明 | 特定年の投資推薦 |
| 屋外10m風速 | 気象観測の参考 | 牛体付近風速の代用 |

データは `data/` に配置する。観測、派生値、デモ、将来シナリオ、利用者入力を同じ種類の値として扱わない。

## 機能追加の順序

### 進捗 — 2026-07-16時点

- [x] 1. 初期モデルケースを編集可能にする
- [x] 2. 根拠を画面へ反映する — 目標風速、3頭／台、法定耐用年数7年、出典区分、均等配置、第1期のデモ仮定を表示
- [x] 3. THIを背景情報として表示する — 期間別中央値・モデル範囲と各案の電力費へ接続済み
- [ ] 4. 現場観察を支援する — 牛・ファン選択は可能、現場確認項目との接続が残る
- [x] 5. 適応経路を表示する — 第1期、見直し年、観測項目、残る全数追加案を表示
- [x] 6. 詳細な採算確認を補助機能にする — 主表示5項目と折りたたみの標準仮定へ接続済み

### 1. 初期モデルケースを編集可能にする

第1期の比較台数を、画面上で変更できるようにする。ファン台数・新規カバー推計牛・状態カードを同期させる。

### 2. 根拠を画面へ反映する

全酪連資料の目標風速、3頭／台、耐用年数を、値・単位・出典・区分つきで表示する。既存ファンの均等配置と第1期5台はデモ仮定として区別する。

### 3. THIを背景情報として表示する

保存済みCMIP6複数モデルから、2026〜2030年と2031〜2034年のTHI対象日数を背景カードへ表示する。各モデルの期間内年平均を先に求め、モデル間の中央値と最小〜最大だけを示す。標準の1対象日24時間を介して各追加案の年間電力費へ接続するが、ファン台数と投資年は変更しない。

### 4. 現場観察を支援する

未カバー推計の牛床を選び、次を確認できるようにする。

- 牛体付近の風速が2m/s以上か
- 牛の呼吸や起立状態に暑熱の兆候がないか
- 既存ファンが狙った牛床へ向き、稼働しているか

初期入力は増やさず、結果を見た後に必要な情報だけを確認する。

### 5. 適応経路を表示する

第1期の後に、次を表示する。

- 今すること
- 次に観測すること
- 次に動く条件
- 残る選択肢

次に動く条件は、利用者・専門家が設定する。システムが根拠なく自動生成しない。

### 6. 詳細な採算確認を補助機能にする

必要な場合だけ、全酪連の7年標準計算を折りたたみで確認できるようにする。主画面の結論やCTAには使わない。

## 画面分割の方針

現在は計算と表示の統合確認を優先し、一画面に全機能を置いている。財務・気候・次の行動までの黄金経路が安定した後は、次の3段階へ分ける。

1. 条件確認 — 千葉市、搾乳牛頭数、牛床列数、既存ファン数
2. 牛舎比較 — 現在の不足、第1期、頭数目安まで追加、年度別状態
3. 費用と成立条件 — 導入費、電気代、回収条件、標準仮定、次の確認事項

画面を分けても入力値と選択中の案を失わず、前後移動でき、各画面上部で現在条件を短く確認できることを着手条件とする。現在の垂直スライスを壊す全面改修は行わない。

## 実装しないこと

- 自動おすすめ投資年
- 根拠不明な閾値による判定
- 将来データの機械的な外挿
- DB、認証、補助金判定、債務・税務の精密モデル
- 本格Choice Horizonの年数判定

## 現在のデータ状態

`data/README.md` にデータ配置と利用区分を記載している。気象庁・千葉の2020〜2025年日別観測と、同一CMIP6モデルの2020〜2025年基準を事前生成済みである。現在の画面は、利用者入力と明示的なモデルケースに加え、保存済み将来気候を運転日数と電力費の背景にだけ使う。直近観測基準とのモデル別差分接続と+1℃／+2℃デモは未接続である。
