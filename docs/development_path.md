# Dairy Horizon 開発方針と機能追加の道筋

## 目的

Dairy Horizonは、投資判断や見積依頼を自動化する画面ではない。

> 農家が細かい数字を知らなくても、今の牛舎で何が不足しているか、小さく始めるならどの状態を比較するか、次に何を確認すべきかを理解できる適応投資経路ナビゲーター

を目指す。

## 現在の導線

1. 地域、搾乳牛頭数、牛床列数、既存ファン数を入力する。
2. 牛舎で、カバー済み牛と未カバー推計牛を確認する。
3. 現在、第1期、全数整備を切り替えて比較する。
4. 次に現場で確認すべきことを一つ表示する。
5. 画面末尾で、標準仮定と出典区分を確認する。

主画面では投資判断、見積依頼、投資年の自動推薦を行わない。

## 第1期の扱い

第1期の5台は、技術基準や最適解ではない。利用者が一度に全数整備を考えず、比較を始めるための初期モデルケースである。

- 初期値は `min(5, 不足台数)` とする。
- 結果画面で、0台から不足台数まで変更可能にする。
- 5台は `demo_assumption` と表示する。
- 変更時には、追加ファンと新規カバー推計牛を牛舎上で更新する。

一方、全酪連資料の「牛体付近で風速2m/s以上」「3頭につき1台以上」は `industry_guidance` として扱う。

## データの利用方針

Webリクエスト中に外部APIを呼ばない。事前に保存・検証したデータだけを読む。

| データ | 用途 | 用途外 |
|---|---|---|
| 千葉の2024年実測THI | 暑い日の参考情報 | 年間台数や投資年の決定 |
| +1℃／+2℃のデモ暑熱ストレス | 暑熱負荷の説明 | 将来の確定予報 |
| 2025〜2034年CMIP6複数モデル | 5年単位の背景と不確実性の説明 | 特定年の投資推薦 |
| 屋外10m風速 | 気象観測の参考 | 牛体付近風速の代用 |

データは `data/` に配置する。観測、派生値、デモ、将来シナリオ、利用者入力を同じ種類の値として扱わない。

## 機能追加の順序

### 1. 初期モデルケースを編集可能にする

第1期の比較台数を、画面上で変更できるようにする。ファン台数・新規カバー推計牛・状態カードを同期させる。

### 2. 根拠を画面へ反映する

全酪連資料の目標風速、3頭／台、耐用年数を、値・単位・出典・区分つきで表示する。既存ファンの均等配置と第1期5台はデモ仮定として区別する。

### 3. THIを背景情報として表示する

千葉の実測THIを小さな「暑い日の参考」カードへ表示する。将来シナリオは折りたたみの詳細に置き、5年単位の中央値と範囲だけを示す。

### 4. 現場観察を支援する

未カバー推計の牛床を選び、次を確認できるようにする。

- 牛体付近の風速が2m/s以上か
- 牛の呼吸や起立状態に暑熱の兆候がないか
- 既存ファンが狙った牛床へ向き、稼働しているか

初期入力は増やさず、結果を見た後に必要な情報だけを確認する。

### 5. 適応経路を表示する

第1期の後に、次を表示する。

- 今すること
- 次に観測すること
- 次に動く条件
- 残る選択肢

次に動く条件は、利用者・専門家が設定する。システムが根拠なく自動生成しない。

### 6. 詳細な採算確認を補助機能にする

必要な場合だけ、全酪連の7年標準計算を折りたたみで確認できるようにする。主画面の結論やCTAには使わない。

## 実装しないこと

- 自動おすすめ投資年
- 根拠不明な閾値による判定
- 将来データの機械的な外挿
- DB、認証、補助金判定、債務・税務の精密モデル
- 本格Choice Horizonの年数判定

## 現在のデータ状態

`data/README.md` にデータ配置と利用区分を記載している。現在の画面は、利用者入力と明示的なモデルケースだけを計算に使う。THI・観測・将来気候データは整理済みだが、画面へはまだ接続していない。
